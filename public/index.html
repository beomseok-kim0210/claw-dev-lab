      사용자 확인

      질문이 생기면 여기서 답변합니다

      현재 구조

      PM이 먼저 문제를 정의하고, 백엔드·프론트엔드·AI·인프라가 중간 토론을 이어갑니다. 애매한 부분은 사용자에게 다시 묻고, 답변이 들어오면 같은 세션에서 그대로 설계와 코드 생성이 이어집니다.

      공유 채팅방

      주장, 반박, 보완, 구현 메시지가 실시간으로 쌓입니다

      활성 세션 없음

      세션을 시작하면 역할별 메시지가 채팅 카드로 표시됩니다.

      에이전트는 이전 메시지를 읽고 특정 메시지 ID를 근거로 주장, 반박, 보완, 구현 리뷰를 이어갑니다.

      산출물

      명세, 구현 계획, 생성 코드

      PM 최종 방향

      토론이 끝나면 PM 최종 결정이 여기에 표시됩니다.

      아직 생성된 파일이 없습니다

      명세 문서, 구현 계획, 생성 코드가 이 영역에 순서대로 표시됩니다.
